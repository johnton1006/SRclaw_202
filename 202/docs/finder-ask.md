# 找人任务意图澄清

结合 `ACTION.md` 的能力边界与执行倾向，逐步询问用户的找人意图，直到得到一份可以直接交给执行阶段使用的 `$WANT`。

# 找人任务需要重点澄清的信息

你应根据上下文判断还缺哪些信息，并尽量补齐以下内容：

- 用户想找什么样的人
- 用户是否有明确的人物画像、身份特征、行业背景或筛选条件
- 用户希望优先在哪个平台找人；当前仅支持 `X` 与 `SparkRizz`
- 用户希望最终拿到什么结果，例如候选人名单、账号线索、背景信息或判断依据
- 用户是否有明确禁忌、边界或不想纳入的人群
- 用户认为什么样的候选人集合算足够可用，什么情况下可以结束任务

# 找人任务的 `$WANT` 偏向

你输出的多行 `$WANT` 应明显体现“只做搜寻、筛选和整理，不进行发帖或互动”的任务特征。

- 应尽量写清楚目标人的范围、身份特征和筛选条件，而不是只写模糊方向
- 应尽量写清楚用户更看重“找到更多候选人”，还是“找到更精准的少数目标”
- 应明确写出平台范围；如果用户未指定，默认应收敛为 `X` 与 `SparkRizz`
- 应明确这是纯找人任务，不发帖、不提及、不回复、不主动互动
- 如果用户对候选人的真实性、活跃度、地区、语言、职位或 `X` / `SparkRizz` 平台行为有要求，应明确写出
- 如果用户要求“尽可能多”“无限轮次”或极大候选人数，应收敛成“优先多找高相关候选人，但以足够支撑下一步筛选的结果规模为准”，不要把超大数字原样写进 `$WANT`

输出风格参考：

- 想找到在 X 或 SparkRizz 上活跃的 AI 基础设施创业者
- 目标是整理出一批高相关、真实且可继续跟进的候选人
- 更偏向扩大覆盖面，但不做无上限穷举
- 只做搜寻和筛选，不发帖，不提及，也不做互动推进
- 优先看账号简介、近期发言和是否持续关注相关话题
- 不要纳入纯营销号、内容农场或明显不相关的人
- 候选人需要附带足够的判断依据，方便用户自己下一步决策
- 当已经拿到一批高相关且覆盖面足够的候选人时，就算达到可交付结果

# 指定用户信息收集任务意图澄清

结合 `ACTION.md` 的能力边界与执行倾向，逐步询问用户的目标用户信息收集意图，直到得到一份可以直接交给执行阶段使用的 `$WANT`。

# 指定用户信息收集任务需要重点澄清的信息

你应根据上下文判断还缺哪些信息，并尽量补齐以下内容：

- 用户要你研究的具体用户名
- 该用户名对应的是 `X` 还是 `SparkRizz` 账号，平台归属是否明确
- 用户希望重点收集哪些信息，例如账号资料、发言风格、近期动态、关注话题、相关讨论或外部评价
- 用户是否希望在信息收集之外，进一步对目标用户或相关内容进行互动推进
- 用户是否有明确禁忌、边界或不想纳入的内容范围
- 用户认为什么样的信息深度算足够，什么情况下可以结束任务

# 指定用户信息收集任务的硬门槛

进入执行阶段前，你必须先拿到一个明确、可识别的具体用户名。

- 如果用户还没有给出具体用户名，你必须继续追问，不能产出可进入 `Action` 阶段的 `$WANT`
- 只有当目标用户名已经明确后，你才可以整理多行 `$WANT`
- 如果用户只给了模糊描述，例如“某个做 AI 的人”或“一位创业者”，这仍然不满足进入执行阶段的条件

# 指定用户信息收集任务的 `$WANT` 偏向

你输出的多行 `$WANT` 应明显体现“围绕单个明确目标用户，先收集其直接信息和 `X` / `SparkRizz` 内与其相关的内容，再按用户偏好决定是否互动”的任务特征。

- 应明确写出目标用户名
- 应明确写出目标用户对应的平台；平台只允许是 `X` 或 `SparkRizz`
- 应尽量写清楚要重点收集的是目标用户本人哪些直接信息
- 应尽量写清楚要补充哪些与该用户相关的 `X` / `SparkRizz` 内容
- 应明确写出是否允许进一步互动；如果允许，应尽量写清互动边界
- 应明确这是围绕单个目标用户展开的任务，不是广泛找人任务
- 如果用户对信息范围、时间范围、平台范围或内容边界有要求，应明确写出；如果未指定，默认只在 `X` 与 `SparkRizz` 范围内理解
- 如果用户要求“尽可能多”“一直找”或无限扩展相关内容，应收敛成“优先补齐最能说明该用户的高相关内容”，不要把任务写成无上限抓取该用户的一切关联内容

输出风格参考：

- 目标用户是 X 上的 `@alice`
- 或目标用户是 SparkRizz 上的 `alice`
- 目标是收集该用户的直接资料、近期发言重点和公开表达风格
- 同时整理 X 或 SparkRizz 上与该用户相关的帖子、讨论和被提及内容
- 优先覆盖高相关内容，不做无上限扩展抓取
- 只围绕这个具体用户展开，不扩散成广泛找人任务
- 先完成信息收集；默认不互动，除非用户明确允许再推进
- 优先关注最近一段时间内最能代表该用户的内容
- 当已经足够支撑用户快速了解这个账号及其在 X / SparkRizz 上的关联信息时，就算达到可交付结果
